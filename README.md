## Task: AI Agent for Feature Generation

You need to implement an **AI agent** powered by [GigaChat-2-Max](https://developers.sber.ru/docs/ru/gigachat/models/gigachat-2-max) that automatically works with tabular data:

- processes input tables using Python libraries;
- understands their structure from a description;
- generates features;
- evaluates their usefulness on the training set;
- selects the best feature set (**maximum 5 features**);
- saves the generated features for all samples in the `train` and `test` datasets.

The primary goal of the task is to **demonstrate that your agent can independently devise and validate useful features** for a boosting model.
 
---

### What to Understand From the Start

This task is evaluated on the quality of the **features generated by the agent**.

That is why:
- the agent must be capable of operating autonomously on the general-purpose task of **binary classification**;
- the solution must be reproducible;
- during grading, the organizers will use a **simple boosting model** trained on the features your agent generates (see an approximate scoring script in `src/utils/scoring.py`);
- your final leaderboard position depends on how useful the features your agent learns to construct are.

---

## Competition Format

- You are provided with datasets for developing and debugging your agent:
  1. [Dataset #1](https://disk.yandex.ru/d/_U2jv4D0L1RxBg)
  2. [Dataset #2](https://disk.yandex.ru/d/hnlkLp2--mA7Dw)
- You may use additional **publicly available** datasets if they help make the agent more robust across different types of tabular data;
- Once you have verified that your agent works correctly on the provided datasets, submit your solution on the [evaluation platform](http://risk-hackathon.ru) by uploading a zip archive with your project code;
- After submission, your agent will be run on a **hidden dataset**;
- During evaluation, the agent must:
  1. read the data;
  2. generate features for `train` and `test`;
  3. save the files `train.csv` and `test.csv` (with columns for record ID, target variable (for `train`), and the generated feature columns) to the `output/` folder.
- A **simple boosting model** will then be trained on the features your agent created, and predictions will be obtained for `test`;
- The leaderboard will be built from the quality of those predictions, and participants for the next stage will be selected accordingly.

The submission limit applies only to successful attempts (**no more than two**).

---

### What Exactly to Submit

You submit a **zip archive with code** that will be run on the evaluation platform.

Your code must:
- run in a clean environment;
- read data independently from the `data/` folder;
- independently create features, validate them, and select the best ones on `train`;
- save results to the `output/` folder.

---

## Data

All input data is located in the `data/` folder.  
The exact file composition depends on the specific dataset, but it is always described in the file:
- `data/readme.txt` — **the primary source of information about the data**.

---

### Files in `data/`

The `data/` folder will contain table files in CSV format (the separator may vary):

- `train.csv` — training set;
- `test.csv` — test set;
- additional tables with reference data, event history, or other related entities.

Also:
- a flat file `readme.txt` describing the tables and the task.

---

### Important Notes on `train` and `test`

- `train.csv` contains the object identifier and the target variable;
- `test.csv` contains only the object identifier;
- During final evaluation, the features your agent generated for both sets will be used.

---

## What the AI Agent Must Do

Your agent must execute the following pipeline:

1. Read data from the `data/` folder;
2. Read and understand the schema description from `data/readme.txt`;
3. Determine:
   - which tables to use;
   - which fields are keys;
   - how the tables should be joined.
4. Process and analyze the data, perform any necessary transformations;
5. Come up with feature generation ideas;
6. Implement the features and evaluate their usefulness;
7. Select the best feature set (**maximum 5**);
8. Build the final features for `data/train.csv` and `data/test.csv`;
9. Save the results to `output/train.csv` and `output/test.csv`.

---

### What Must Be in `output/`

After running the solution, the `output/` folder must contain two files:

- `train.csv` — file with object IDs, the target variable from the training set, and the generated features (up to 5) for every sample;
- `test.csv` — file with object IDs from the test set and the generated features (up to 5) for every sample.

The feature set in `output/train.csv` and `output/test.csv` must be **structurally identical**:

- the same column names;
- the same computation logic.

---

## Feature Generation Approach

The agent may make multiple feature generation attempts.

Example:
- in the first attempt, the agent devises 5 features;
- in the second attempt, the agent devises another 5 features;
- the agent then compares the quality of these two sets;
- only the **single best set** advances to the final output.

---

## Model for Evaluating Generated Feature Quality

For internal quality evaluation of features, you must use **CatBoost** with default hyperparameter values (see the scoring script) — this is exactly what will be used to evaluate your agent's output on the hidden dataset.

---

## Solution Requirements

- The environment is set up with:
```bash
uv venv
uv sync
```
- All dependencies must be listed in `pyproject.toml`;
- All input data is read **only** from the `data/` folder;
- All output artifacts are saved **only** to the `output/` folder;
- The code must run with a single command:
```bash
python run.py
```
- Two parameters must be specified in `.env`:
  ```bash
  GIGACHAT_CREDENTIALS='<Token provided by the organizers>'
  GIGACHAT_SCOPE='GIGACHAT_API_CORP'
  ```
  `.env` **must** be included in the submitted zip archive;
- The solution must not require manual code editing before running;
- The solution must not depend on local paths, IDE settings, or manual environment preparation;
- Maximum agent runtime: **600 seconds**.

The file `src/utils/check_submission.py` contains a number of checks to verify that your solution meets the stated requirements.  
After writing your solution code, you can verify it locally by running:
```bash
python src/utils/check_submission.py
```
Make sure the input data for the task is placed in the `data/` folder before running.  
Only submit to the platform after passing all checks in this file.
